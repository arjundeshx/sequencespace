When DeepMind released AlphaFold 2 in 2020, the structural biology community collectively exhaled. A problem that had sat on the CASP leaderboard for three decades — predicting a protein's 3D structure from its amino acid sequence — had been, for most practical purposes, solved. The 2024 Nobel Prize in Chemistry made that official.

AlphaFold 3 pushed further: multimers, nucleic acids, small molecule ligands, covalent modifications. The scope is genuinely breathtaking. But as I've spent the last few months integrating AF3 predictions into an actual drug discovery pipeline, I've developed a more complicated view. We haven't solved structural biology. We've solved one very important subproblem of it.

What AF3 Actually Does (and Doesn't Do)

The core output of AlphaFold 3 is a single predicted structure — or more precisely, a sample from an implicit distribution over structures. This is a subtle but important point. Proteins are not static objects; they breathe, flex, undergo large-scale conformational changes, and often require disorder to function.

"The structure is the vocabulary. The dynamics are the language."

For well-folded globular proteins where you need a starting point for docking or mutagenesis design, AF3 is transformative. But for the approximately 30% of human proteins that are intrinsically disordered, or for understanding allosteric mechanisms, you need more than a single coordinate set.

The Disorder Problem

Intrinsically disordered regions (IDRs) pose a specific challenge. AF3's pLDDT confidence score is a reasonable proxy for disorder — low pLDDT correlates with experimentally confirmed disorder — but the structural predictions themselves are not meaningful for these regions. They're essentially noise with a low confidence flag.

💡 Practical Note
When using AF3 predictions as input to downstream analyses, always mask out residues with pLDDT < 50 before computing structural features. Treating low-confidence regions as reliable structure is a common and consequential mistake.

Running AF3 Locally: A Setup Guide

The open-weight AF3 release requires some setup. Here's the pipeline I use for batch predictions on a protein family of interest: 

# Install af3 and dependencies
# Requires CUDA 12+, ~2TB storage for databases

import json
from pathlib import Path
from alphafold3 import run_inference

# Build input JSON for a protein of interest
def make_af3_input(name: str, sequence: str) -> dict:
    return {
        "name": name,
        "sequences": [{
            "protein": {
                "id": "A",
                "sequence": sequence
            }
        }],
        "modelSeeds": ["42", "137", "999"],  # multiple seeds = ensemble
        "dialect": "alphafold3",
        "version": 1
    }

sequences = {
    "TP53_human": "MEEPQSDPSVEPPLSQETFSDLWKLLPENNVLSPLPSQAMDDLMLSPDDIEQWFTEDP...",
    "PCNA_human":  "MFEARLVQGSILKKVLEALKDLINEACWDISSSGVNLQSMDSSHVSLVQLTLRSEGFDF..."
}

for name, seq in sequences.items():
    inp = make_af3_input(name, seq)
    Path(f"/data/af3_inputs/{name}.json").write_text(json.dumps(inp))

What Comes After Structure?

The most exciting frontier, in my view, is not better static structure prediction — it's using predicted structures as priors for dynamic simulation. Tools like OpenMM and GROMACS can now be seeded with AF3 coordinates to run molecular dynamics, and the combination is genuinely powerful.

There's also a growing ecosystem of structure-aware machine learning models — GVP, EquiFold, and others — that take structural representations as input and predict function, binding affinity, or mutational effects. The bottleneck is no longer getting structures; it's knowing what to do with them.

My Current Workflow

For a recent project predicting the effect of point mutations on binding stability, my pipeline looked like this: AF3 prediction → ensemble generation with 5 random seeds → per-residue confidence filtering → Rosetta ddG calculation → ESM-1v zero-shot mutant scoring → ensemble averaging. The combination outperformed any single method.

Conclusion

AlphaFold 3 is a genuinely transformative tool. But transformative tools don't end fields — they open new ones. The protein folding problem, narrowly defined, is largely solved. The problems of protein dynamics, disorder, and function remain wide open.

If you're a computationalist just getting into structural biology, this is an exciting time. The infrastructure is finally good enough that you can ask biological questions rather than engineering questions. Don't waste that by treating AF3 as an oracle.
